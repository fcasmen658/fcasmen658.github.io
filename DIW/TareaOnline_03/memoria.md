## Actividad 1: Análisis de accesibilidad y usabilidad

### 1. Comprueba manualmente los siguientes aspectos de accesibilidad de tu web (realiza capturas de las diferentes comprobaciones):

#### Perceptible:
✓ ¿Es la página responsive?
✓ ¿Es el texto también responsive?
✓ ¿Cómo verían esta web los usuarios con daltonismo?
✓ ¿Proporciona alternativas textuales para el contenido no textual?
✓ Si hay videos, ¿tienen estos subtítulos?

#### Operable:
✓ ¿Es la página accesible mediante el uso únicamente de teclado? ¿Tiene una navegación con el tabulador adecuada?
✓ ¿Proporciona suficiente tiempo para leer el contenido antes de que caduque la sesión?
✓ Investiga como las personas ciegas visualizan una página web y que las dificultades encuentran. ¿Qué recomendaciones harías para maximizar la accesibilidad del sitio webs ante usuarios ciegos?

##### Comprensible
✓ ¿Cómo es el uso del lenguaje de la página web?
✓ ¿Notifica errores del usuario de forma adecuada?

##### Robusto
✓ ¿Es la navegación igual entre dos navegadores diferentes?

## Conclusiones

### Objetivos Alcanzados
✓ Landing page completa y responsiva  
✓ Páginas de ficha de producto con Grid Layout  
✓ Página de búsqueda con Flexbox  
✓ Integración de Bootstrap en 3 o más componentes  
✓ Modo oscuro funcional con accesibilidad

✓ Header y footer reutilizables  
✓ Tipografía fluida y responsive  
✓ Estados hover/focus implementados  
✓ Variables CSS para mantenibilidad  

### Aprendizajes Clave
1. **Variables CSS:** Facilitan mantenimiento y cambios globales
2. **Grid vs Flexbox:** Cada uno tiene su uso óptimo (Grid para layouts 2D, Flex para 1D)
3. **Bootstrap:** Acelera desarrollo pero requiere personalización para identidad única
4. **Modo Oscuro:** Requiere planificación desde el inicio con variables semánticas
5. **Accesibilidad:** No es opcional, debe integrarse desde el diseño
